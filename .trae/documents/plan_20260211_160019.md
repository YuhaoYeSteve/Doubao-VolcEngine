我建议采用 **“动态徽章 + 自动更新列表”** 的组合方案，既能直观展示项目活跃度，又能保留详细的提交记录，同时不破坏现有的精美排版。

### 📅 实施计划

#### 1. ✨ 添加动态状态徽章 (Badges)
在 `README.md` 标题下方添加一行实时的可视化徽章，让访客一眼看到项目的活跃状态。
- **Last Commit**: 显示最后一次提交的时间。
- **Commit Activity**: 显示最近一个月的提交频率趋势图。
- **Repo Size**: 显示项目代码体积。

#### 2. 🤖 配置自动更新的“最近提交”列表
为了补充人工维护的“更新日志”，我将在 README 底部添加一个自动化的 `最近提交 (Auto)` 区域。
- **动作**: 创建 GitHub Action (`.github/workflows/update-readme.yml`)。
- **机制**: 每次 `push` 代码时，自动抓取最新的 5 条 commit 记录，并格式化为列表更新到 README 中。
- **优势**: 无需手动维护，实时反映开发进度。

#### 3. 📝 修改 README.md 结构
- 在顶部插入徽章代码。
- 在底部添加自动化区域的占位符 `<!--START_SECTION:recent_commits-->`。
